# ゲーム一覧プロジェクト 制作ルール

このリポジトリ（ゲーム集）に**ゲームを追加・改修するときに必ず守る決まり**をまとめたものです。
新しいゲームを作るときは、まずこのファイルを読んでから着手してください。

---

## 0. 基本方針

- 各ゲームは原則**1つのHTMLファイルで完結**（`game/games/<name>.html`、または大きいものは `game/<日本語名>/`）。
- 画面は**ダークテーマ＋ゴールド系アクセント**で統一。左上に「← 一覧に戻る」リンクを置く。
- スマホ・PC両対応（タッチ操作とキーボード/クリックの両方）。
- 追加したら必ず**ハブ（`index.html`）に登録**し、**人数フィルターのタグ**を設定する（→ 第6章）。

---

## 1. オフライン対戦のルール

- **オフライン対戦＝CPUとの対戦**とする。
- **CPUの強さは3段階以上**から選べるようにする（例：`弱 / 普通 / 強`）。
  - 弱＝ほぼランダム、普通＝定石・基本手、強＝先読み/評価関数つき、が目安。
- 1人用パズル（2048・テトリス・数独・ソリティア・マインスイーパー・インベーダー等）はCPU対戦の対象外。

---

## 2. オンライン対戦のルール

接続は PeerJS を使った P2P。**部屋番号は4桁**で全ゲーム共通の番号空間（`ngroom-` 接頭辞）。

### 2-1. 入室の流れ
1. **ホスト**が「部屋を作成」して**4桁の部屋番号を発行**する。
2. **ゲスト**はロビー（ハブまたはゲーム内のオンラインパネル）で**部屋番号を入力して参加**する。
   - 番号だけで、ホストが選んだゲームに自動で入る（別ゲームのページなら自動で正しいページへ転送）。

### 2-2. 開始
- ホストは**ゲストが集まってから**、**CPUの人数と強さを選択**して試合を開始する。
  - N人対戦（最大4人）では、人間＋CPUの合計が最小人数〜最大人数に収まるように。

### 2-3. 途中退出と代行
- 人間が**途中退出したら、CPUが操作を引き継ぐ**（その人が復帰するまで）。
  - ホストが代行を担当する。ホスト自身が抜けた場合は、残ったプレイヤーの中でホストを引き継ぐ（ホストマイグレーション）。

### 2-4. 復帰（再接続）
- 途中退出した人間は、**前回の対局に復帰できる**。
- **前回途中退出した場合のみ**、ロビーのオンライン対戦欄に「**🔄 中断した対局に再接続**」ボタンを表示する（通常時は出さない）。
- 復帰時は**その時点の盤面（スナップショット）から再開**する（抜けていた間のログ再生はしない）。

### 2-5. 盤面の向き
- 将棋・チャイニーズチェッカー・オセロ・五目並べ・ガイスター・はさみ将棋・バックギャモン等、**盤に向きがあるゲームは、ホストとゲストで向きを調節**する。
  - 各プレイヤーから見て**自分の駒（手前）が下**になるようにする。

### 2-6. 隠し情報
- 手札などの隠し情報は**各自の席からの見え方でレンダリング**する（相手の手札は裏向き）。
  - 通信は「同じ乱数シードで両者が同じ局面を計算する」ロックステップ方式。カジュアルな信頼前提（チート対策は厳密には行わない）。

---

## 3. 手番表示のルール（重要）

**「自分の番」か「相手の番」かが一目で分かるようにする。**

- 手番テキストの**フォント色を変える**などして、**自分の番を目立たせる**。
  - 自分の番＝緑系で太字・強調（バッジは脈動アニメ）。相手の番＝控えめなグレー。
- プレイヤーは**先攻から順に 赤・青・緑・黄（P1〜P4）** で色分けする。
  - 配色: `赤 #e0473a / 青 #3e87d6 / 緑 #4ec05e / 黄 #e0c34a`（`PCOL` 配列）。
  - **プレイヤー名**と**「誰の番です」表記**の両方に、その色を反映する。
- 先攻・後攻（親・起家など）は**ランダムで決定**する（オンラインは共有シードで両者一致させる）。

---

## 4. AI評価値のルール

- **必勝法・最善手がある（完全情報の）オフラインゲーム**には、参考として**AI評価値（ヒント/アシスト）**を付ける。
  - 例：オセロ・五目並べ・将棋などの「AIアシスタント」。各手の良し悪しや推奨手を表示する。
  - 三目並べのように完全に解けるものは、最善手で負けないCPU＋（任意で）評価表示。
- 表示は**ON/OFF切り替え可能**にし、初心者の学習補助とする（対人の本気対戦では切れるように）。

## 5. 共有ファイルと実装の使い分け

| 種別 | ファイル | 用途 |
|---|---|---|
| N人オンライン（最大4人・CPU補填あり） | `shared/online-room.js` | ホスト権限の星形中継・ロックステップ。CPU人数選択、退出代行、再接続、ホスト引き継ぎを内蔵 |
| 2人オンライン（クリック中継） | `shared/online-core.js` | 盤に向きのある2人用。要素インデックスでクリックを中継。`flip` で盤を180°回転 |
| ロビー・部屋番号ルーティング | `shared/lobby.js` | 4桁番号の名前空間と `GAME_PATHS`（ゲームID→パス） |

- **新しいオンラインゲームを足したら、`shared/lobby.js` の `GAME_PATHS` に `ゲームID: "パス"` を必ず追加**する（番号からの自動転送に必要）。
- 2人用で `online-core.js` の `flip` を使うときは、**両クライアントで同数の要素になるよう**にする（ゲストだけ `<style>` を足さない）。クリック中継は document 全体の要素インデックスに依存するため。
- 手番判定マーク（`hostMark`/`guestMark`、例 `先手`/`後手`）は、**対局中つねにステータス文言に含める**こと（含めないと手番が判定できず動かなくなる）。

---

## 6. ハブ（`index.html`）への登録手順

新しいゲームを追加したら、`index.html` に以下を行う：

1. **ゲームカードは `.grid` の一番末尾に追加**する（既存の順番は変えない）。
   - アイコン・名前・説明・「プレイする →」リンクを設定する。
   - オンライン対応なら `🌐 オンライン` タグも付ける（`...?online=1&_v=2`）。
2. **人数フィルターのタグを登録**：`TAGS` マップに `"ゲーム名": ["solo" | "duo" | "multi"]`。
   - `solo`=1人、`duo`=1VS1、`multi`=3人以上。
3. オンライン対応なら、**「部屋を作成」モーダルにも入口を追加**（`...?online=1&_v=2&create=1`）。

## 7. 「← 一覧に戻る」リンクの実装（全ゲーム必須）

**すべてのゲームファイルに必ず設置する。**

### 配置場所・スタイル

`<body>` の最初の要素として、以下のHTMLをそのままコピーして使う。

```html
<a href="../index.html" style="position:fixed;top:10px;left:10px;z-index:99999;background:rgba(0,0,0,0.55);color:#fff;text-decoration:none;font:600 13px/1 'Segoe UI',sans-serif;padding:8px 13px;border-radius:999px;border:1px solid rgba(255,255,255,0.25)">← 一覧に戻る</a>
```

- `position: fixed` で画面左上に常時表示する（ゲーム中でも隠れない）。
- `z-index: 99999` で最前面に出す。
- サブディレクトリ (`game/<日本語名>/`) に置く場合は `href="../../index.html"` に変更する。

## 8. スマホ対応（全ゲーム必須）

**すべてのゲームはスマホ（タッチ操作）でプレイできなければならない。**

### 必須要件

1. **`<meta name="viewport">`** を `<head>` に入れる。
   ```html
   <meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no">
   ```

2. **レイアウト**：画面幅に合わせてキャンバス・盤面が縮小するようにする。
   - Canvas ゲーム：`T（タイルサイズ）= Math.floor(Math.min(window.innerWidth - 余白, 最大幅) / 列数)` のように可変サイズにする。
   - HTML盤面：`max-width: 100%; overflow: hidden;` で横スクロールを出さない。

3. **タッチ操作**：クリックが必要な箇所はタッチでも同等に動くようにする。
   - `touchstart` / `touchend` を `click` と併用する（または `pointer` イベントで統一）。
   - スマホ向けの**仮想Dパッド・ボタン**が必要なゲームは画面下部に配置する（例：ボンバーマンの矢印パッド）。
   - `e.preventDefault()` でダブルタップズームや画面スクロールを防ぐ。

4. **文字・ボタンサイズ**：最小タップ領域は `44px × 44px` 以上を目安にする。

5. **`touch-action: none`**：Canvas や操作エリアに設定してブラウザのデフォルトジェスチャーを無効化する。

### チェック方法

- Chrome DevTools の「デバイスツールバー」（Ctrl+Shift+M）でスマホ表示を確認する。
- 幅 390px（iPhone 14 相当）で崩れないことを確認する。

---

## 9. キャッシュ対策（GitHub Pages 配信のため）

- ハブからゲームへのリンク・転送には**バージョンクエリ `&_v=N`** を付け、古いHTMLがキャッシュから読まれないようにする。
- 共有JS（`online-core.js` / `online-room.js` / `lobby.js`）の読み込みは **`?v=N`** を付け、更新時に番号を上げる。
- 仕様変更後は両端末で一度**ハード更新**（PC: `Ctrl+Shift+R` ／ スマホ: タブを閉じて開き直す）すると最新が読まれる。

## 10. チェックリスト（ゲーム追加・改修時）

- [ ] オフラインはCPU対戦、強さ**3段階以上**
- [ ] オンライン：ホスト部屋作成 → ゲスト番号入力で参加
- [ ] ホストがCPU人数・強さを選んで開始
- [ ] 退出時はCPUが代行、復帰可能（ロビーに**中断時のみ**再接続ボタン）
- [ ] 向きのあるゲームはホスト/ゲストで向き調整（自分が手前）
- [ ] 手番が一目で分かる（自分の番を強調、P1〜P4を赤青緑黄）
- [ ] 先攻後攻はランダム
- [ ] 完全情報ゲームはAI評価値（ON/OFF可）
- [ ] `GAME_PATHS`・ハブのカード・フィルタータグ・キャッシュクエリを登録
- [ ] 「← 一覧に戻る」リンクを設置（`position:fixed` で左上、`z-index:99999`）
- [ ] スマホ対応：`viewport` メタタグ・レイアウト縮小・タッチ操作・仮想ボタン（必要な場合）
